A credit-data analyst's job when providers disagree: figure out why, produce an auditable reconciliation, and flag which assessment is stale or driven by outdated inputs.

Deterministic Logic · No LLM

Every divergence, decoded by pure C#

One notch ladder, one decomposer, and six red-flag rules — all reproducible, cited, and LLM-free. The model only narrates what the code already proved.

① Notch Ladder
Maps every provider's letter — Moody's Aaa…C, DBRS (high/mid/low), S&P/Fitch — onto a canonical 1–21 integer. Tolerant of outlook / CreditWatch decorations; NR / WR / D (withdrawn, not-rated, default) never crash — they become coverage gaps.
② Divergence Decomposer
Splits each notch gap into Weighting + Input + Methodology buckets that reconstruct the gap exactly — an invariant pinned by a property test over 200 random pairs. When data is letter-only, it says so honestly instead of faking a precise split.
// Invariant — the waterfall always ties out
assert weighting + input + methodology == notchGap;
③ Red-Flag Engine · six deterministic rules
HIGH STALE_INPUT — a rating predates the latest filing by > 45 days.
HIGH IG_HY_BOUNDARY — providers straddle the investment-grade / high-yield line.
MED OUTLIER_PROVIDER — the lone provider ≥ 3 notches from the peer median.
MED METHODOLOGY_CONFLICT — a wide residual gap not explained by timing or the outlier.
MED MISSING_COVERAGE — a provider publishes no rating.
LOW PROVIDER_UNDER_REVIEW — a provider on CreditWatch may re-rate.
Severity is fixed by the rule, not the model. Each flag shows its verbatim C# rule text and cited source rows on click, never double-reports a gap, and is pinned by 260 tests — reproducible on stage, every time.
